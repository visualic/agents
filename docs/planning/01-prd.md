# SkillForge PRD (제품 요구사항 명세)

## MVP 캡슐
1. 목표: Claude Code의 스킬/에이전트/오케스트레이션을 패턴 기반으로 쉽게 만드는 개인용 데스크톱 도구
2. 페르소나: Claude Code를 적극 활용하는 개발자 (본인)
3. 핵심 기능: FEAT-1 패턴 라이브러리, FEAT-2 가이드 대화, FEAT-3 코드 생성/내보내기
4. 성공 지표 (노스스타): 스킬/에이전트 제작 소요 시간 50% 단축
5. 입력 지표: 패턴 라이브러리 등록 수, 생성된 스킬/에이전트 수
6. 비기능 요구: 로컬 전용 (인터넷 없이 패턴 관리 가능)
7. Out-of-scope: 멀티 유저, 클라우드 동기화, 마켓플레이스
8. Top 리스크: 패턴의 품질과 다양성이 충분하지 않으면 도구 가치가 떨어짐
9. 완화/실험: bkit 등 기존 스킬을 분석하여 초기 패턴 10개 이상 확보
10. 다음 단계: 패턴 분석 및 구조 정의

---

## 1. 제품 개요

- **서비스명**: SkillForge (가칭)
- **핵심 가치**: "구조를 모르겠다"는 장벽을 패턴과 가이드로 해소
- **타겟 사용자**: 본인 (L4 전문가, 현업 개발자 + PM/PO 경험)
- **플랫폼**: Electron 데스크톱 앱

### 배경
Claude Code의 스킬, 에이전트, 오케스트레이션은 생산성을 극대화하는 핵심 도구이지만, 직접 만들 때 진입 장벽이 매우 높다:
- 시작점을 모르겠다 (어떤 구조로 마크다운을 작성해야 하는지)
- 구조에 자신이 없다 (잘 만든 건지 확인할 수 없음)
- 베스트 프랙티스를 모르겠다 (참고할 패턴이 부족)

### 해결 전략
잘 만든 패턴을 수집/분류하고, 그것을 기반으로 대화하며 커스터마이즈해서 자신만의 스킬/에이전트/오케스트레이션을 만든다.

---

## 2. 핵심 기능

### FEAT-1: 패턴 라이브러리
- **설명**: 잘 만든 스킬/에이전트/오케스트레이션의 구조적 패턴을 수집, 분류, 조합
- **Why**: 시작점과 구조를 모르는 문제를 해결 - 좋은 예시가 있으면 변형이 가능
- **우선순위**: MVP 핵심 (1순위)
- **상세**:
  - 패턴 카드 그리드로 시각적 탐색
  - 구조 미리보기 다이어그램 포함
  - 유형별 분류 (스킬/에이전트/오케스트레이션)
  - 태그 기반 검색/필터링
  - 패턴 상세에서 코드 보기/수정/조합

### FEAT-2: 가이드 대화
- **설명**: 5단계 하이브리드 방식으로 스킬/에이전트를 점진적으로 정교하게 설계
- **Why**: 복잡한 구조를 혼자 설계하기 어려움 - AI와 대화하며 단계적으로 완성
- **우선순위**: MVP 핵심 (2순위)
- **5단계 프로세스**:
  1. 유형/목적 정의 ("어떤 스킬을 만들 건가요?")
  2. 트리거 설계 ("언제 활성화될까요?")
  3. 구조 설계 ("레퍼런스 파일은? 제약조건은?")
  4. 도구/권한 ("어떤 도구를 쓸 수 있나요?")
  5. 미리보기/확인

### FEAT-3: 코드 생성 + 내보내기
- **설명**: 설계 결과를 Claude Code 호환 마크다운 파일로 생성하고 프로젝트에 배치
- **Why**: 최종 산출물이 Claude Code에서 바로 사용 가능해야 도구의 가치가 완성됨
- **우선순위**: MVP 핵심 (3순위)
- **상세**:
  - SKILL.md, 에이전트.md 등 표준 형식으로 생성
  - `.claude/skills/` 또는 `.claude/agents/` 디렉토리에 직접 내보내기
  - 레퍼런스 파일 자동 생성

## 3. 사용자 스토리

- As a Claude Code 사용자, I want 패턴 라이브러리에서 좋은 예시를 탐색하고 싶다, so that 스킬/에이전트 구조의 감을 잡을 수 있다
- As a Claude Code 사용자, I want AI와 대화하며 스킬을 단계적으로 설계하고 싶다, so that 혼자서는 놓칠 수 있는 디테일을 잡을 수 있다
- As a Claude Code 사용자, I want 설계 결과를 바로 Claude Code에 적용하고 싶다, so that 만든 스킬/에이전트를 즉시 사용할 수 있다
- As a Claude Code 사용자, I want 내가 만든 것들을 "내 작업실"에서 관리하고 싶다, so that 이전에 만든 것들을 재활용할 수 있다
- As a Claude Code 사용자, I want 여러 패턴을 조합해서 새로운 것을 만들고 싶다, so that 복잡한 오케스트레이션도 쉽게 구성할 수 있다

## 4. 제약 조건

- **시간**: 개인 프로젝트, 기한 없음
- **예산**: 0 (Anthropic API 비용은 Claude CLI 기존 구독으로 해결)
- **인력**: 1인 (본인)
- **기술 제약**: Electron + React, 로컬 전용

## 5. MVP 범위

### Phase 0 (MVP)
- 패턴 라이브러리 (카드 그리드 + 상세 보기)
- 가이드 대화 (5단계 하이브리드, Claude CLI 연동)
- 코드 생성 + 내보내기
- 내 작업실 (생성 이력 관리)

### Phase 1 (확장)
- 패턴 조합 기능
- 패턴 업로드/등록
- 외부 패턴 수집 자동화

### Out-of-scope
- 다른 사용자 지원
- 클라우드 동기화
- 마켓플레이스
- 모바일 지원
