# SkillForge TRD (기술 요구사항 명세)

## 1. 기술 스택

### 프론트엔드
- **Electron**: 데스크톱 앱 프레임워크
- **React**: UI 라이브러리
- **TypeScript**: 타입 안정성
- **Tailwind CSS**: 유틸리티 퍼스트 스타일링

### 백엔드 (Electron Main Process)
- **Node.js**: Electron 내장
- **fs/path**: 로컬 파일 시스템 접근
- **child_process**: Claude CLI 호출

### 데이터 저장
- **SQLite** (better-sqlite3): 패턴 메타데이터, 검색 인덱스
- **로컬 파일 시스템**: 패턴 원본 마크다운 파일, 생성된 스킬/에이전트 파일

### AI 연동
- **Claude CLI**: `claude` 명령어를 child_process로 호출
- 별도 API 키 관리 불필요 (기존 Claude Code 설정 활용)

## 2. 아키텍처

### 전체 구조
```
┌─────────────────────────────────────────┐
│          Electron App                    │
│  ┌───────────────────────────────────┐  │
│  │     Renderer Process (React)       │  │
│  │  ┌──────┐ ┌──────┐ ┌──────────┐  │  │
│  │  │패턴   │ │가이드 │ │내 작업실  │  │  │
│  │  │라이브러리│ │대화  │ │         │  │  │
│  │  └──────┘ └──────┘ └──────────┘  │  │
│  └───────────────┬───────────────────┘  │
│                  │ IPC                   │
│  ┌───────────────┴───────────────────┐  │
│  │     Main Process (Node.js)         │  │
│  │  ┌──────┐ ┌──────┐ ┌──────────┐  │  │
│  │  │SQLite │ │파일   │ │Claude    │  │  │
│  │  │      │ │시스템 │ │CLI 연동  │  │  │
│  │  └──────┘ └──────┘ └──────────┘  │  │
│  └────────────────────────────────────┘  │
└─────────────────────────────────────────┘
         │              │
    ┌────┴────┐   ┌────┴────┐
    │ SQLite  │   │ .claude/ │
    │ DB      │   │ 디렉토리 │
    └─────────┘   └─────────┘
```

### 패턴
- **IPC 통신**: Renderer ↔ Main 간 Electron IPC
- **파일 기반 패턴 저장**: 각 패턴은 마크다운 파일로 저장, SQLite는 메타데이터/인덱스만
- **Claude CLI 래퍼**: 가이드 대화 시 Claude CLI를 스트리밍 호출

---

## 3. 보안 요구사항

- **인증**: 불필요 (개인 로컬 앱)
- **인가**: 불필요
- **파일 시스템 접근**: Electron의 로컬 파일 접근 권한 필요
- **Claude CLI**: 기존 설치된 CLI의 인증 정보 활용 (별도 관리 없음)

---

## 4. 성능 요구사항

- **앱 시작**: 3초 이내
- **패턴 검색**: 100ms 이내 (SQLite 인덱스)
- **카드 렌더링**: 패턴 100개 기준 부드러운 스크롤
- **Claude CLI 응답**: CLI 자체 성능에 의존 (스트리밍으로 UX 개선)
- **파일 내보내기**: 즉시 (로컬 파일 시스템 쓰기)

---

## 5. 개발 환경

- **Node.js**: 20+ LTS
